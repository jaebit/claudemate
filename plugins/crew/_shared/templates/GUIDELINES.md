## Workflow Rules

### 1. Initialization

- Always ensure `.caw/` environment exists before starting work
- Use `/crew:init` to initialize, `/crew:init --reset` to reinitialize
- Check `/crew:dashboard` for current workflow state

### 2. Tidy First Principle

Follow Kent Beck's Tidy First methodology:

| Type | Emoji | Description | Commit Separately |
|------|-------|-------------|-------------------|
| **Tidy** | 🧹 | Structural changes (no behavior change) | Yes |
| **Build** | 🔨 | Behavioral changes (new features, fixes) | Yes |

**Never mix Tidy and Build changes in the same commit.**

### 3. Planning Before Execution

1. `/crew:start "task description"` - Create structured plan
2. Review `task_plan.md` before proceeding
3. `/crew:next` - Execute steps one by one
4. `/crew:review` - Review completed work

### 4. Parallel Execution

- **Phase Deps**: Phases with same dependency can run in parallel
- **Step Deps**: Steps with no blocking deps can run in parallel
- Use `/crew:next --worktree phase N` for parallel execution
