Python의 **GIL(Global Interpreter Lock)**은 CPython 인터프리터에서 한 번에 하나의 스레드만 Python 바이트코드를 실행할 수 있도록 보장하는 뮤텍스(Mutex)입니다. 이 메커니즘이 멀티스레딩 성능에 미치는 영향은 다음과 같습니다.

### 1. CPU 집약적(CPU-bound) 작업에서의 성능 저하
*   **병렬성 제한:** 여러 개의 코어가 있더라도 GIL 때문에 실제로 코어 하나에서만 작업이 수행됩니다.
*   **오버헤드 발생:** 스레드 간 컨텍스트 스위칭(Context Switching) 비용과 GIL을 획득/해제하려는 경쟁으로 인해, 싱글 스레드로 실행할 때보다 오히려 속도가 더 느려질 수 있습니다.

### 2. I/O 집약적(I/O-bound) 작업에서의 효율성
*   **성능 이점:** 파일 입출력, 네트워크 요청 등 I/O 작업 중에는 스레드가 대기 상태로 들어가며 GIL을 해제합니다.
*   **병행성 유지:** 이 덕분에 다른 스레드가 CPU를 사용할 수 있어, I/O 작업이 많은 애플리케이션에서는 멀티스레딩이 여전히 유효한 성능 향상을 제공합니다.

### 3. GIL이 존재하는 이유
*   **참조 계수(Reference Counting):** Python의 메모리 관리 방식인 참조 계수가 스레드로부터 안전(Thread-safe)하지 않기 때문에, 이를 보호하기 위해 도입되었습니다.
*   **단순성 및 하위 호환성:** C로 작성된 수많은 확장 모듈(C Extension)들이 GIL의 존재를 가정하고 설계되어 있어 제거가 복잡합니다.

### 4. 성능 한계 극복 방법
*   **Multiprocessing:** `threading` 대신 `multiprocessing` 모듈을 사용하면 별도의 프로세스를 생성하여 각 프로세스마다 고유한 GIL을 가지게 되므로 실제 병렬 처리가 가능합니다.
*   **C 확장 모듈:** NumPy나 Pandas 같은 라이브러리는 내부 연산을 C로 수행하며, 연산 중에 GIL을 해제하여 멀티 코어를 활용합니다.
*   **다른 인터프리터:** GIL이 없는 PyPy(STM), Jython, IronPython 등을 고려할 수 있습니다. 최근 Python 3.13에서는 실험적으로 GIL을 선택적으로 비활성화할 수 있는 기능(free-threaded Python)이 추가되었습니다.
